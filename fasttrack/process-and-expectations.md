# <a name="process-and-expectations"></a>流程與預期值

在此主題中，涵蓋了如何使用 FastTrack 取得協助、協助如何提供、取得成功體驗的秘訣，以及瞭解 FastTrack 和公司的重要責任。

## <a name="engaging-fasttrack"></a>使用 FastTrack

取得 FastTrack 專家的協助  
  
您可以取得 FastTrack 專家協助的兩個位置：

  - 透過 Microsoft 系統管理中心-您的租使用者管理員登入 Microsoft 系統管理中心，然後按一下 **需要協助嗎？** 小工具，接著依照要求程式操作。
  - 透過 FastTrack 網站–您或您的合作夥伴透過 FastTrack 網站向貴公司取得協助。

若要直接透過 FastTrack 網站要求協助：

1.  登入 [FastTrack 網站](https://go.microsoft.com/fwlink/?linkid=780698)。
2.  從登陸頁面上方的 **快速動作** 選取 **使用 Microsoft 365 要求協助**。
3.  填妥**使用 Microsoft 365 要求協助** 表單。

若要讓您的合作夥伴以您的 FastTrack 網站替您取得協助，請讓您的合作夥伴：

1.  登入 [FastTrack 網站](https://go.microsoft.com/fwlink/?linkid=780698)。
2.  從登陸頁面上方的 **快速動作** 選取 **使用 Microsoft 365 要求協助**。
3.  輸入客戶名稱、網域或 TPID 來搜尋您的客戶。
4.  從搜尋結果中選取您的客戶。
5.  填妥**使用 Microsoft 365 要求協助** 表單。

## <a name="fasttrack-expectations"></a>FastTrack 預期值

FastTrack 提供有關如何使用 Microsoft 365 的端對端指導方針。 在規劃成功發行時，您可以取得遠端指導方針和最佳做法。

### <a name="meeting-rhythms"></a>會議節奏

您可以透過下列技術交談、研討會和檢查點會議，與 FastTrack 專家遠端工作，包括：

  - **技術交談和研討會：** 這些提供主題深入研究和最佳做法，以加速部署 Microsoft 365 服務。 這些會議通常提供可用的文件、軟體工具和系統管理員主控台的概覽。 標準持續時間：60分鐘。
  - **檢查點會議：** 每週或雙週接觸點，以確保您的部署處於執行狀態。 標準持續時間：30分鐘。

### <a name="expected-outcomes-and-key-milestones"></a>預期的結果和關鍵里程碑 

FastTrack 提供部署最佳做法和修正步驟，以便讓貴組織能夠使用 Microsoft 365。

 **FastTrack 可預期的目標：**

  - 我們會與您一起合作以建立補救計畫，將您的來源環境達到符合成功上線和移轉的最基本需求。
  - 我們為核心登入、身分識別整合及服務啟用活動提供堅實的基礎。

**一般預定里程碑包括：**

  - 啟動會議–瞭解程式、排程、工具和團隊活動。
  - 評估–開發修正檢查清單。
  - 安全性討論–根據建議的 Microsoft 安全性最佳做法，規劃安全性戰略。
  - 修正檢查–檢查任何修正專案。
  - 成功研討會：建立行動計畫的工具、範本和指南。
  - 檢查點會議–會議紀要和排程。
  - 啟用活動-完成服務啟用。
  - 移轉速度研討會（選用-僅限 Microsoft 365）-移轉準備就緒。

如需詳細資訊，請參閱 [Microsoft 365 客戶成功案例](https://www.microsoft.com/microsoft-365/customer-stories?rtc=1)。

### <a name="customer-success-tips"></a>客戶成功秘訣

您對 FastTrack 的協助準備是獲取平順且成功的程式的關鍵。 在使用 FastTrack 前，請務必建立：

  - 所需的日期（您打算部署服務的時間）。
  - 意向（您計畫要啟動的服務）。
  - 商務原因（您計畫向使用者推出服務的原因）。
  - 成功擁有者（負責確保專案成功的人員）。

我們也建議您考慮並識別下列專案：

  - 風險承擔者–行政主管和擁護者的招聘和權力。
  - 案例–成功條件的優先順序和定義。
  - 注意–通訊活動的實施和規劃啟動活動。
  - 訓練–教導使用者並完成支援服務資源。
